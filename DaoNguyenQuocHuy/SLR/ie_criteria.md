# IE Criteria — LLM-based REST API Test Generation and Testing

**Thành viên:** Võ Gia Phú

**RQ:** Đối với REST API có OpenAPI specification (P), LLM tự động sinh API test cases bao phủ edge cases và error codes (I) so với manual test design (C) có đạt endpoint coverage ≥90% và phát hiện thêm ≥2 real bugs không?

**PICO:**

- **P (Population):** REST APIs with OpenAPI Specifications
- **I (Intervention):** LLM-generated API test cases covering edge cases and error codes
- **C (Comparison):** Manual test design / hand-written test cases
- **O (Outcome):**
  - Endpoint Coverage ≥ 90%
  - Detect ≥ 2 real bugs

---

## Inclusion Criteria (IC) – paper PHẢI có đủ tất cả

| Mã | Tiêu chí |
|-----|----------|
| **IC1** | Paper is written in English |
| **IC2** | Published from 2020 onwards |
| **IC3** | Peer-reviewed journal, conference paper, or complete research preprint |
| **IC4** | Focuses on REST API testing, OpenAPI testing, or API test generation |
| **IC5** | Uses LLMs or Generative AI for test generation |
| **IC6** | Includes empirical evaluation or experimental results |

---

## Exclusion Criteria (EC) – loại nếu BẤT KỲ điều kiện nào đúng

| Mã | Tiêu chí |
|-----|----------|
| **EC1** | Duplicate paper |
| **EC2** | Full text or PDF is unavailable |
| **EC3** | Poster, extended abstract, tutorial, or presentation only |
| **EC4** | Focuses only on API specification/documentation without testing |
| **EC5** | Focuses only on test execution, maintenance, naming, or benchmarking without proposing a testing approach |
| **EC6** | Does not use LLMs or Generative AI |
